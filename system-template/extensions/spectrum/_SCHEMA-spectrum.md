## 〇-A、為什麼恰好是三個頻段？（設計者備註 2026-05-29）

### 數學約束：Nyquist 極限與濾波器過渡帶

理論上可以無限細分（1,339 場景 → 最多 ~669 個頻率分量）。但 Butterworth 濾波器的**過渡帶**不是無限窄的——4 階濾波器的過渡帶約為截止頻率的 20-30%。當相鄰頻段的間距小於過渡帶時，頻段之間會**洩漏**（一個頻段的信號污染另一個）。實際可區分的頻段約 10-15 個，不是無限。

### 語義約束：頻段必須對應可感知的敘事尺度

| 頻段 | 週期 | 敘事對應 | 唯物史觀層次 |
|------|------|---------|------------|
| 長波 >50 | 幾年～一生 | 世界觀轉變、意識形態演化 | **結構**（生產方式、階級關係） |
| 中波 10-50 | 一個篇章 | 情節弧線、階段性情緒 | **局勢**（具體歷史時刻、力量對比） |
| 短波 <10 | 幾個場景 | 事件、對話、瞬間反應 | **事件**（個別行動、偶然性） |

多於三個頻段（如 7 個或 15 個），多出來的頻段在語義上沒有對應的敘事概念——「15-30 場景週期」在故事中是什麼？既不是篇章也不是事件，是一個模糊的中間地帶。AI 被注入這些坐標時無法有效加權。

三個頻段恰好對應托洛茨基在《俄國革命史》中的三層分析：結構—局勢—事件。這不是巧合，是唯物史觀方法論的自然收斂。

### 實用約束：Prompt 信息密度

三個頻段的坐標約 80-120 tokens。十個頻段會占用 300-400 tokens，擠壓文本證據的空間，且 AI 的注意力機制對相近尺度的坐標產生稀釋效應。

### 結論

> **三個頻段是在 Nyquist 極限（數學）、敘事語義（方法論）、Prompt 密度（工程）三個約束下的最優解。邊際效益在三個之後急劇遞減。**

## 〇、命名由來

> 為什麼叫「頻譜制」？在無線電工程中，頻譜分析同時監聽所有頻段（長波/中波/短波），知道「哪個頻率在什麼時間、以什麼強度活躍」。敘事頻譜制將這個概念遷移到文本分析：長波對應世界觀層面的慢變量（>50 場景週期），中波對應篇章結構（10-50 場景），短波對應場景級事件（<10 場景）。每個場景不是被孤立分析，而是被放置在三維頻譜坐標中——它知道自己「在整個故事弧線中的位置」。

## 一、核心概念

### 1.1 三個頻段

| 頻段 | 週期 | 敘事對應 | 物理類比 | 功能 |
|------|------|---------|---------|------|
| **長波 (LW)** | >50 場景 | 世界觀基調、意識形態轉變、角色一生心理弧線 | 板塊運動——極慢但決定一切 | **定位**：這個場景在整個故事中的歷史位置 |
| **中波 (MW)** | 10-50 場景 | 篇章結構、情節弧線、階段性情緒變化 | 季風——規律性循環 | **導航**：當前處於篇章的哪個階段（上升/頂點/下降） |
| **短波 (SW)** | <10 場景 | 單場景事件、對話情緒、微觀反應 | 陣風——突發脈衝 | **通信**：場景內的即時事件 |

### 1.2 干涉模式（⚠️ 信號，不是噪音）

| 模式 | 物理類比 | 敘事意義 | 審計優先級 |
|------|---------|---------|:--------:|
| **constructive** | 波峰疊加，振幅增大 | 情緒與結構共振——高置信度 | P2（自動通過） |
| **destructive** | 波峰遇波谷，互相抵消 | 轉折點、異常事件、或 mood 誤標 | **P0（必審）** |
| **boundary** | 信號太弱 | 篇章邊界，缺乏上下文 | P1（抽檢） |
| **harmonic** | 整數倍關係 | 結構性重複／母題循環 | P2（自動標記） |

### 1.3 為什麼頻譜層必須「完全本地、零 API」

> 頻譜計算的數學基礎（FFT、Butterworth 濾波）在 1,339 個浮點數上的運行時間 < 100ms。如果在這個規模上調用雲端 API，不僅成本荒謬，更重要的是**頻譜層必須在每次審計前重建**（因為 mood 標籤可能被修改）。本地計算是唯一合理的選擇。

## 二、計算方法

### 2.1 核心算法（偽代碼）

```python
import numpy as np
from scipy import signal

# 1. 從 compiled-scenes.json 提取情緒數值序列
mood_values = [mood_to_val(s['mood']) for s in arc_scenes]

# 2. 多尺度分解
# 長波：>50 場景週期 → 低通濾波（截止頻率 0.02，即 1/50）
b_l, a_l = signal.butter(4, 0.02, 'low')
longwave = signal.filtfilt(b_l, a_l, mood_values)

# 中波：10-50 場景週期 → 帶通濾波（0.02–0.1，即 1/50–1/10）
b_m, a_m = signal.butter(4, [0.02, 0.1], 'band')
midwave = signal.filtfilt(b_m, a_m, mood_values)

# 短波：<10 場景 → 殘差（原始信號 − 長波 − 中波）
shortwave = np.array(mood_values) - longwave - midwave

# 3. 週期檢測（FFT）
fft = np.fft.rfft(mood_values)
dominant_period = 1.0 / np.argmax(np.abs(fft[1:]))  # 跳過 DC 分量

# 4. 時頻定位（STFT，可選）
f, t, Zxx = signal.stft(mood_values, fs=1.0, nperseg=20)
```

## 五、適用前提（⚠️ 必讀）

> 1. **你的數據必須是連續時間序列**。離散條目（如知識庫）沒有頻譜可分析。
> 2. **你需要一個可數值化的指標**。mood 是自然選擇，但也可以是用其他指標（對話密度、場景長度、角色出場數）。
> 3. **數值映射必須通過客觀性校準**。這是整個頻譜層的單點故障。

## 六、設計備忘錄（給未來的自己）

> **2026-05-29**：頻譜層的靈感來自一個簡單的觀察——LLM 的注意力機制是 flat 的。Transformer 知道 token A 和 token B 有關，但不知道 token A 在小說的開端而 token B 在尾聲。這不是模型的錯——「時間」這個維度根本不在輸入中。頻譜層用最便宜的方式（本地 FFT）補上了這個維度。
>
> **為什麼用 Butterworth 而不是小波**：小波變換更精確（同時保留時間和頻率分辨率），但 Butterworth 濾波器足夠用於「這個場景處於長波上升段第 31.4% 位置」這種精度。關鍵是要簡單——越簡單的算法越容易被理解、被信任、被 debug。
>
> **阿基里斯腱**：mood→數值的映射。如果三個人映射出三個不同的頻譜，頻譜層就死了。這時不要留戀——降級為純三進制模式。頻譜層是錦上添花，不是雪中送炭。
