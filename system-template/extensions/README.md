## 〇、為什麼是這三個？

這三個擴展來自同一個洞見：**認知系統的瓶頸不在 AI 的能力，在組織形式。**

| 擴展 | 解決的問題 | 對應的哲學範疇 |
|------|-----------|-------------|
| **三進制 (Ternary)** | 二進制輸出（✅/❌）強行消解矛盾 → 改為 Asserted / Contested / Unknown 三態 | **辯證法**：矛盾的統一與鬥爭——矛盾不應被調和，應被顯性化 |
| **蘇維埃 (Soviet)** | 中央節點獨裁決策導致級聯失真 → 改為基層自治 + 矛盾上報 + 多版本輸出 | **民主集中制**：基層民主節點自主判斷，中央不裁決而繪製張力圖譜 |
| **頻譜制 (Spectrum)** | AI 沒有「時間」概念，憑感覺做判斷 → 用本地 FFT 計算長波/中波/短波坐標 | **歷史唯物主義**：物質條件（頻譜坐標）制約意識（語義判斷），具體地分析具體的條件 |
| **漣漪角色 (Construct)** | 角色索引只收錄已出場者 → 從已知角色漣漪推導「缺席的應然存在者」 | **結構因果性**：制度/血緣/歷史結構中「理應存在」的角色，不是憑空發明 |

### 三者的層級關係

```
頻譜制（Layer 3）────────────── 物質約束層
  │  提供精確的時間坐標（長波/中波/短波相位）
  │  完全本地計算，零 API 成本
  │
  ▼
三進制（Layer 2）────────────── 語義判斷層
  │  在頻譜坐標上進行 Asserted / Contested / Unknown 判斷
  │  矛盾被顯性化而非消化
  │
  ▼
蘇維埃（組織原則）────────────── 架構治理層
  │  基層節點自主輸出三元判斷
  │  中層節點繪製張力圖譜（不做「整合」）
  │  頂層節點輸出多版本敘事（不做「裁決」）
  │  各系統實例平行自治，通過清晰合約協作
```

### 三者的關係不是「上級指揮下級」

```
錯誤理解（軍隊模式）：        正確理解（蘇維埃模式）：
                              
  Layer 3                     [頻譜] ──物質約束──▶ [三進制]
    │                           │                    │
    ▼                           │        ┌───────────┘
  Layer 2                       │        ▼
    │                         [蘇維埃]◀──矛盾顯性化
    ▼                           │
  Layer 1                       ▼
                             [二進制] ←──剛性驗證（YAML/Schema/Seq）
                              
                             三層之間互相制約，沒有誰指揮誰
                             頻譜約束語義，但不決定語義
                             語義防止頻譜淪為機械決定論
                             蘇維埃組織防止兩者脫離文本事實
                             [漣漪] ──結構因果──▶ 補全缺席者
```

## 一、三進制 (Ternary)：認知狀態的三態

### 1.1 設計意圖（為什麼需要這個）

**問題**：傳統 LLM 輸出單一答案（「李奧很憤怒」），即便文本中存在同等強度的相反證據（「他忽然笑了，伸手替她撥開頭髮」）。這導致矛盾被「消化」在敘事衝動中——AI 會說「他的憤怒中夾雜著複雜的溫情」，把矛盾包裝成「深度」。

**解決**：每個判斷必須攜帶其**認知狀態**——不是「這個判斷是對的」，而是「這個判斷目前的證據狀態是確定的／有爭議的／未知的」。

### 1.2 三態定義

| 狀態 | 含義 | 觸發條件 |
|------|------|---------|
| **Asserted** | 文本證據充足，判斷可靠 | 錨定文本 ≥ 3 處，支持度 ≥ 0.7，無同等強度反證 |
| **Contested** | 文本中存在支持與反對的同等強度證據 | 正反證據強度差 < 0.3，或存在明確的反證文本 |
| **Unknown** | 文本信息不足，或需要跨篇章上下文 | 錨定文本 < 2 處，或處於篇章邊界 |

### 1.3 輸出格式

```yaml
# 二進制輸出（舊——禁止使用）
mood: "疏離"
confidence: 0.85

# 三進制輸出（新——必須使用）
mood:
  primary: "疏離"
  valence: 0.82
  status: contested       # asserted | contested | unknown
  anchors:
    - scene: 47
      line: 1284
      quote: "他退後一步，像是被燙傷"
      support: 0.91
  dissent:
    - position: "溫情/依戀"
      evidence: [scene-128, scene-203]
      strength: 0.74
      unresolved: true    # 矛盾被顯性化，而非調和
  synthesis: null          # 基層強制留空——因果解釋留給中層
```

### 1.4 關鍵約束（⚠️ 不可違反）

> **基層節點被禁止輸出「連續敘事」**。它只能輸出觀察（primary）、把握（valence）、反證（dissent）、矛盾標記（status）。因果解釋和成長軌跡留給中層。
>
> **理由**：級聯失真的根源是基層節點在證據不足時用「敘事衝動」填補空白。禁止基層「講故事」是防止虛假確定性的第一道防線。AI 最危險的時刻不是它說「我不知道」，而是它說「這是辯證的統一」——那通常意味著它剛剛消化了一個不該被消化的矛盾。

### 1.5 適用數據形態

| 數據形態 | 可否使用 | 說明 |
|---------|:------:|------|
| 連續時間序列（場景、編年史、日記） | ✅ | 三態判斷 + 頻譜坐標 |
| 離散條目（知識庫、術語詞典） | ✅ | 三態判斷（無頻譜坐標） |
| 結構化數據（配置、Schema） | ❌ | 不需要三元，二進制驗證已足夠 |

### 1.6 包含文件

| 文件 | 用途 |
|------|------|
| `_SCHEMA-ternary.md` | 三元 Frontmatter 字段定義（可 merge 入主 `_SCHEMA.md`） |
| `_audit-ternary.py` | 調用 AI API，對指定條目輸出三態判斷 |

## 二、蘇維埃 (Soviet)：民主節點組織原則

### 2.1 設計意圖（為什麼需要這個）

**問題**：傳統分層架構中，上級節點「整合」下級節點的輸出，強行消解矛盾以產出單一答案。這本質上是**認知獨裁**——中央節點有權力否決基層節點的 contested 標記。結果是級聯失真：每一層都在替下一層「做決定」，錯誤以冪次放大。

**解決**：採用蘇維埃式的民主節點組織——每個節點自主判斷、矛盾不上報消解而是上報顯性化、中央不裁決而繪製張力圖譜。

### 2.2 蘇維埃五原則

| # | 蘇維埃原則 | 架構對應 | 具體實現 |
|---|-----------|---------|---------|
| 1 | **基層蘇維埃自治** | 每個 `system-template` 實例完全自包含 | 自己的 Schema、自己的 compiled DB、自己的 Prefix、自己的 TASKLOG。不需中央資料庫。 |
| 2 | **矛盾不上報消解** | 當兩個實例對同一事實有不同標註時，矛盾被標記為 contested | 在 AGENTS.md 路由表中記錄「系統 A 標 asserted，系統 B 標 contested」，不做「跨系統一致性修正」 |
| 3 | **中央只繪圖，不裁決** | AGENTS.md 只是路由表 + 張力圖譜 | 告訴 Agent「這裡有分歧」，不告訴 Agent「誰是對的」 |
| 4 | **多版本輸出** | 頂層（人類）收到多個版本的敘事 | 每個版本標註前提假設、被忽略的證據、選擇代價 |
| 5 | **罷免權** | 人工始終擁有最終裁定權 | 可以接受 contested 為常態、選擇一個版本、或要求更多證據 |

### 2.3 蘇維埃 vs 傳統分層

| | 傳統分層架構（軍隊模式） | 蘇維埃架構（代表會模式） |
|---|---|---|
| **基層輸出** | 單一答案 + 置信度 | 三態認知單元 + 反證 + 文本錨定 |
| **中層任務** | 整合 → 產出更高層次的抽象 | 繪製張力圖譜 → 標註斷層與盲區 |
| **頂層任務** | 最終答案 | 多版本敘事生成 |
| **矛盾處理** | 被消化（編碼成「成長」「辯證統一」） | 被顯性化（contested 節點、unresolved_knots） |
| **人類角色** | 被動接受答案 | 主動選擇版本 |
| **級聯失真風險** | 高（每層都在替下層做決定） | 低（決定權保留在數據結構中） |
| **比喻** | 將軍下令，士兵執行 | 每個蘇維埃自主決策，中央只是協調秘書處 |

### 2.4 關鍵約束（⚠️ 不可違反）

> **中層節點被禁止輸出「整合後的單一敘事」**。它的唯一任務是繪製張力圖譜——告訴上層：哪裡平坦（asserted）、哪裡有斷層（contested）、哪裡是盲區（unknown）。
>
> **頂層節點被禁止輸出「唯一正確答案」**。它必須輸出至少兩個版本 + 一個 unknown 選項，並透明展示每個版本的取捨。
>
> **理由**：如果中層做了「整合」，它就替頂層做了決定。如果頂層做了「唯一答案」，它就替人類做了決定。蘇維埃模式的核心是**決定權不下放**——每一層只增加訊息，不減少選項。

### 2.5 蘇維埃模式的適用邊界

> ⚠️ **蘇維埃模式不適合所有場景**。當你需要的是確定性答案（「這個 YAML 語法正確嗎？」）時，二進制層（Layer 1）的剛性驗證比民主節點更有效。蘇維埃模式的核心價值在於**語義不確定性的領域**——文學分析、歷史判斷、情感標註——這些地方強求「共識」本身就是暴力。

### 2.6 包含文件

| 文件 | 用途 |
|------|------|
| `_ROUTING-soviet.md` | 蘇維埃式路由模式：多實例平行自治、矛盾上報、多版本輸出的設計模式 |
| `_tension-map.py` | 從多個實例的三態輸出中生成跨系統張力圖譜 |

## 三、頻譜制 (Spectrum)：敘事時間的物質約束

### 3.1 設計意圖（為什麼需要這個）

**問題**：LLM 在分析連續敘事（如 1,339 場景的小說）時，沒有真正的「時間」概念。Transformer 的注意力機制是 flat 的——它知道「這個 token 和那個 token 有關係」，但不知道「第 47 場景在整個故事弧線中處於什麼位置」。結果是 AI「讀完全文後憑感覺說」，把高潮段的情緒誤判給開端段。

**解決**：用本地 NumPy/SciPy 的 FFT 和濾波器，將情緒序列分解為長波（>50 場景週期）、中波（10-50）、短波（<10）三個頻段。每個場景獲得精確的頻譜坐標（相位、振幅、趨勢），注入 AI 的 Prompt 作為「認知 GPS」。

### 3.2 三個頻段的物理意義

| 頻段 | 週期 | 敘事對應 | 物理類比 | 功能 |
|------|------|---------|---------|------|
| **長波 (LW)** | >50 場景 | 世界觀基調、意識形態轉變、角色一生心理弧線 | 板塊運動——極慢但決定一切 | **定位**：這個場景在整個故事中的歷史位置 |
| **中波 (MW)** | 10-50 場景 | 篇章結構、情節弧線、階段性情緒變化 | 季風——規律性循環 | **導航**：當前處於篇章的哪個階段 |
| **短波 (SW)** | <10 場景 | 單場景事件、對話情緒、微觀反應 | 陣風——突發脈衝 | **通信**：場景內的即時事件 |

### 3.3 干涉模式（⚠️ 這是信號，不是噪音）

| 干涉模式 | 物理類比 | 敘事意義 | 審計優先級 |
|---------|---------|---------|:--------:|
| **constructive** | 兩個波峰疊加，振幅增大 | 情緒與結構一致——高置信度 | P2（自動通過） |
| **destructive** | 波峰遇到波谷，互相抵消 | 轉折點、異常事件、或 mood 誤標——**需要審計** | **P0（必審）** |
| **boundary** | 信號太弱，無法判斷 | 篇章邊界，缺乏上下文——標記為 unknown | P1（抽檢） |
| **harmonic** | 一個波是另一個的整數倍 | 結構性重複／母題循環 | P2（自動標記） |

### 3.4 關鍵約束（⚠️ 不可違反）

> **頻譜坐標是「物質約束」，不是「決定因素」**。它告訴 AI 「你現在處於長波上升段第 31.4% 位置」，但不告訴 AI 「這裡應該是憤怒」。判斷仍由 AI 根據文本證據做出——但現在它知道自己在哪裡。
>
> **頻譜層完全本地運行，零 API 成本**。1,339 場景的 FFT + 濾波 < 100ms。
>
> **頻譜層要求輸入是連續時間序列且有可數值化的指標**。這是整個架構的單點故障——如果 mood→數值的映射是主觀的，頻譜輸出的不是敘事結構而是映射者的審美偏好。Phase 1 實驗的首要任務就是驗證這個映射的客觀性。

### 3.5 適用前提（⚠️ 必讀）

> **頻譜層僅適用於連續時間序列數據**。如果你的數據是離散條目（如知識庫），頻譜層不適用——沒有時間軸就沒有「頻率」可分析。在這種情況下，只用三進制即可。
>
> **頻譜層依賴 mood 可數值化**。如果 mood 標籤是純文字（"憤怒"、"悲傷"），需要一個映射表將其轉為數值。這個映射的客觀性是整個架構的阿基里斯腱。建議先用 50 個場景做三人獨立映射 FFT 比較——如果三個人的頻譜一致，則映射客觀；如果不一致，降級為純三進制模式。

### 3.6 包含文件

| 文件 | 用途 |
|------|------|
| `_SCHEMA-spectrum.md` | 頻譜 Frontmatter 字段定義（長波/中波/短波坐標 + 干涉模式） |
| `_build-spectrum.py` | 從 compiled JSON 計算頻譜坐標（純 NumPy/SciPy，零 API） |
| `_validate-spectrum.py` | 干涉模式檢測 + P0-P3 四級分級 |

## 三-A、漣漪角色 (Construct Characters)：缺席的應然存在者

### 3A.1 設計意圖（為什麼需要這個）

**問題**：傳統角色索引只收錄**已出場**的角色。但任何敘事世界都存在「缺席的應然存在者」——他們沒有場景文本可引用，但從歷史邏輯、制度結構、家族關係中可以推導出他們的存在。

例如：威廉二世必然有皇太子（Kronprinz Wilhelm），但 ECC 未寫入；MI5 創始總監 Vernon Kell 必然有副手和核心團隊；聖恩救濟院必然有其他孤兒和修女。

**解決**：從已知角色出發，透過四層漣漪（血緣→制度→歷史→敘事）推導那些「理應存在的人」。這些角色不是「憑空發明」——已知角色是投入歷史池塘的石頭，漣漪圈出缺席者。

### 3A.2 漣漪四層

| 層級 | 推導邏輯 | 置信度 | 範例 |
|:---:|------|:---:|------|
| **R1** | 血緣/家族必然存在 | ≥0.90 | 威廉二世 → 皇太子 |
| **R2** | 官職/機構必有同僚 | ≥0.80 | Vernon Kell → MI5 副手 |
| **R3** | 同時期歷史對應人物 | ≥0.70 | 德皇有太子 → 沙皇的其他大公 |
| **R4** | 故事邏輯需要但未寫 | ≥0.50 | Leo 在伊頓的老師/同學 |

### 3A.3 與既有體系的關係（⚠️ 關鍵約束）

> **Construct 角色不寫入 `index/characters/`**（沒有場景文本可引用）。使用獨立映射檔 `.taskbook/construct-characters/ripple-candidates.yaml`。
>
> **若後續被場景文本確認**：`status: construct` → `status: confirmed`，移入 `character-mappings.yaml`，創建完整的角色檔案。這是從「缺席者」到「在場者」的升級路徑。

### 3A.4 包含文件

| 文件 | 用途 |
|------|------|
| `_SCHEMA-construct.md` | 漣漪角色 Schema 定義、四層推導邏輯、偽代碼、驗證規則 |

```bash
# 1. 複製基礎模板（Layer 1 二進制層——必選）
cp -r system-template/ my-system/

# 2. 按數據形態複製擴展

# 僅需三進制審計（離散條目）：
cp -r system-template/extensions/ternary/ my-system/extensions/

# 連續時間序列 + 頻譜定位：
cp -r system-template/extensions/ternary/ my-system/extensions/
cp -r system-template/extensions/spectrum/ my-system/extensions/

# 多系統協作 + 蘇維埃組織：
cp -r system-template/extensions/ternary/ my-system/extensions/
cp -r system-template/extensions/spectrum/ my-system/extensions/
cp -r system-template/extensions/soviet/ my-system/extensions/

# 補全缺席角色（可獨立使用）：
cp -r system-template/extensions/construct-characters/ my-system/extensions/

# 3. 自訂 _SCHEMA.md（merge 擴展字段定義）

# 4. 執行（按層級順序）
python my-system/scripts/_build-compiled-db.py      # L1: 編譯
python my-system/scripts/_validate-schema.py        # L1: 驗證
python my-system/extensions/spectrum/_build-spectrum.py    # L3: 頻譜（可選）
python my-system/extensions/spectrum/_validate-spectrum.py # L3: 干涉檢測（可選）
python my-system/extensions/ternary/_audit-ternary.py      # L2: 三進制審計（可選）
python my-system/extensions/soviet/_tension-map.py         # 蘇維埃: 張力圖譜（可選）
```

## 六、設計備忘錄（給未來的自己，2026-05-29）

> **核心洞見**：認知系統的瓶頸不在 AI 的能力，在**組織形式**。二進制組織生產「答案」，三進制組織生產「認知地形」。蘇維埃不是政治隱喻，而是對「級聯失真」的結構性解法——如果每層都在替下層做決定，錯誤就會以冪次放大。頻譜不是數學遊戲，而是給 AI 一個它天生缺乏的東西——**時間**。
>
> **為什麼蘇維埃模式在 AI 架構中是合理的**：LLM 的級聯失真（cascading hallucination）不是任何一個節點的「錯誤」，而是分層架構必然的產物——每層被迫輸出確定性答案，不確定性被逐層壓縮，最終產出虛假共識。蘇維埃模式不試圖「消除幻覺」，而是把不確定性**保留在輸出格式中**——contested 和 unknown 是合法的終態，不是「還沒算完」的中間態。
>
> **未解決的問題**：mood→數值的映射是否客觀？這是整個頻譜層的阿基里斯腱。Phase 1 實驗（三個人獨立映射 50 個場景，比較 FFT 輸出）是生死測試。如果三個人的頻譜不一致，頻譜層就降級為 reference-only——頻譜坐標僅供人工參考，不注入 AI Prompt。
>
> **擴展的設計原則是「可獨立拆卸」**：每個擴展只依賴 compiled JSON（這是 Layer 1 的標準輸出），不依賴其他擴展。這確保任何擴展的實驗失敗都不會導致核心基礎設施癱瘓。Layer 1 是唯一的硬依賴。
